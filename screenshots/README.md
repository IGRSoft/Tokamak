# Tokamak demo screenshots

A reusable harness that renders the shared `TokamakDemo` catalog
(`Sources/TokamakDemo/DemoCatalog.swift` — the single source of truth) across multiple
back-ends and writes one PNG per demo into `screenshots/<platform>/<Name>.png`.

Filenames are shared across platforms (`sanitize(name)`: `/` and space → `-`), so the
same demo has the same filename everywhere (e.g. `HStack/VStack` → `HStack-VStack.png`),
enabling cross-platform visual comparison.

These PNGs are surfaced as inline preview tables in
[`docs/progress.md`](../docs/progress.md) — each feature section ends with a **Gallery** table
showing the demo across `mac`/`web`/`ios`/`wasm` (a `—` cell marks a host that does not capture
that demo; see the skip rationale below).

## Regenerate

```sh
Scripts/screenshots/generate.sh all              # every platform the host supports
Scripts/screenshots/generate.sh web mac          # a subset
Scripts/screenshots/generate.sh ios
```

Each platform runs independently; one failing never aborts the others. The driver prints
a PASS/PARTIAL/SKIP summary and exits 0 if any platform produced ≥1 PNG.

## Platforms & prerequisites

| Platform | Tool | Prerequisite | Output | Count | Status |
|----------|------|--------------|--------|-------|--------|
| `mac`    | `swift run ScreenshotNative` (SwiftUI `ImageRenderer`, NSImage) | macOS 13+ | `screenshots/mac/` | ~41 PNGs | must-pass |
| `web`    | `swift run ScreenshotHTML` (ImageRenderer → HTML → Chrome headless) | Google Chrome (or `$CHROME_BIN`) | `screenshots/web/` + `_catalog.json` | ~41 PNGs | must-pass |
| `ios`    | `xcodebuild test` on iOS Simulator (iPhone 17 Pro) | Xcode 15+ | `screenshots/ios/` | ~41 PNGs | must-pass |
| `wasm`   | PackageToJS + static server + Playwright | SwiftWasm SDK + Node.js + Playwright | `screenshots/wasm/` | 52 PNGs (all entries) | best-effort (generated: 52 PNGs) |
| `gtk`    | Docker + GTK4 + Xvfb + ImageMagick | Docker daemon + GTK4 image | `screenshots/gtk/` | ~26 PNGs | best-effort (generated after GTK3→GTK4 attach fix; count varies by demo support) |

### Notes on the `web` path

Tokamak's StaticHTML SSR renders `TokamakCore` views, but on a **macOS host** the demo
catalog compiles against **SwiftUI** (`TokamakShim` → `import SwiftUI`). The two `View`
protocols are distinct, so the SwiftUI catalog cannot be fed into Tokamak's `StaticHTMLRenderer`
on the host, and no Linux/wasm SDK is installed to build it as `TokamakCore`. The host `web`
generator therefore renders each view with SwiftUI `ImageRenderer` and captures it through
**Chrome headless from a standalone HTML page** — a real browser capture, visually faithful to
the view. The authoritative DOM/SSR render is the `wasm` path (shipped artifact in a real browser),
which requires the SwiftWasm toolchain; the gallery is now generated (51 PNGs in
`screenshots/wasm/`).

Eleven demos are skipped on native/web platforms and captured authoritatively via the `wasm` browser path:

- **Canvas** — no static display list under a standalone `ImageRenderer` (TimelineView never ticks offscreen).
- **List, Sidebar** — no intrinsic content size outside a window context.
- **ScrollViewReader** — needs a scroll/window context offscreen.
- **Color, Form & GroupBox, Gauge, LazyVStack/LazyHStack, Label, Slider** — root `ScrollView` is unrasterizable offscreen on macOS `ImageRenderer` (empirically verified).

The `mac`/`web` count is therefore ~41 (= 52 catalog − 11 skipped); `ios` is ~41 (= 52 catalog entries − 11 skipped); `wasm` is 52 (all entries including the 11 skipped on native/web). Exact counts finalized when FN0 regenerates galleries (task #8). This is expected and logged.

Interactive AppKit/UIKit-backed controls (Menu, TabView, PasteButton, SignInWithAppleButton,
HSplitView, VSplitView, and the value selectors) render as shape-based static fallbacks in the
galleries — see `usesStaticControlFallback` / `staticControlFallbackView(for:)` in `DemoCatalog.swift`.

### Best-effort platforms

`wasm` and `gtk` require toolchains that may be absent. When a prerequisite is missing the
phase writes a `SKIPPED.md` (and, for gtk, `UNSUPPORTED.md`) with the exact reason and the
commands to produce the screenshots, then exits 0 — it never blocks the other platforms.

## Single Source of Truth

All generators consume the same `Sources/TokamakDemo/DemoCatalog.swift`, which holds a `@MainActor`
global `demoCatalog: [DemoEntry]` array. Each generator iterates this catalog and renders each
entry to a PNG file using its platform's native renderer (ImageRenderer on native, Chrome on web).
The `_catalog.json` manifest (meta: section, name, file mapping) is generated by the web step and
consumed by downstream tools.

This means the catalog is maintained in **one place**; any new demo added to the catalog
automatically appears in all platforms on the next `generate.sh run`. To add a demo, see the
[Demo Catalog Guide](../docs/DemoCatalog.md).

## Pipeline internals

The shared render loop lives in `Sources/ScreenshotKit/CatalogRender.swift`
(`renderCatalogToPNGs`), which the `mac` and `ios` generators call with a platform PNG encoder
(NSImage / UIImage), while `web` renders via `ImageRenderer` → HTML → Chrome. Each entry is wrapped
by `demoCaptureWrapped` (bounded capture frame + control fallbacks + `\.demoCaptureMode`) before
rendering. Two defense-in-depth gates guard quality:

- **`assessPNG`** — decodes each PNG and rejects blanks/degenerates (too few bytes or distinct
  colors), downgrading them to a logged skip rather than committing an empty image.
- **`containsNosignPlaceholder`** — rejects the AppKit/UIKit "nosign" placeholder that
  `ImageRenderer` paints for unrasterizable controls.
- **`assertNoDuplicateRenders`** — fails if two distinct demo ids produce byte-identical PNGs.

`generate.sh` orchestrates the platforms (`Scripts/screenshots/`), prunes the by-design skips before
the must-pass count gate, and copies the iOS simulator output into `screenshots/ios/`.

## Layout

```
screenshots/
  _catalog.json        # shared manifest {section,name,id,file} (generated by web step)
  README.md
  mac/      <Name>.png (~41 PNGs: 52 catalog − 11 skipped)
  web/      <Name>.png (~41 PNGs: 52 catalog − 11 skipped; _html/ intermediates git-ignored)
  ios/      <Name>.png (~41 PNGs: 52 catalog − 11 skipped)
  wasm/     <Name>.png (52 PNGs: all catalog entries; best-effort, generated)
  gtk/      <Name>.png (~26 distinct demos via TokamakGTK4 + Xvfb) | UNSUPPORTED.md (deny-set/no-content; SKIPPED.md when Docker absent)
```
